## 1. コンセプト & 美的方向性

**何を可視化するか**: この Mac 上で動いている Claude Code セッション群を「AI社員」に見立て、
ピクセルアートの RPG 風オフィス俯瞰図の中でライブ表示するダッシュボード。
セッションの状態 (作業中 / 指示待ち / 承認・質問待ち) がキャラの居場所・吹き出し・スタッツに反映され、
キャラをクリックすると対話でき、テキストで指示も送れる。3秒ごとに全体が更新される。

**トーン = 「cozy な二層構造」**:

1. **UIクロム層** (トップバー・左右サイドバー・カード類):
   クリーム/オフホワイト地のフラットでモダンなカード UI。繊細なボーダー、柔らかい影、角丸。
   情報密度は高いが穏やか。彩度は低く、茶系のインクで統一。
2. **ステージ層** (中央のオフィス俯瞰図):
   対照的にリッチで濃密なピクセルアート。レンガの壁、木目の床、暖色の室内照明、
   深いティール色の会議室、赤いソファの休憩ラウンジ。レトロ RPG (16bit 期の室内マップ) の質感。

この「静かなダッシュボード枠の中に、賑やかなドット絵の世界が窓のように開いている」対比が
このUIの記憶点 (unforgettable な一点)。ステージは**モニター風の黒いベゼル**で額装され、
ページのクリーム地から一段沈んで見える。

キーワード: cozy / 温かみ / 木とクリーム / レトロゲームの職場 / 「会社ごっこ」のユーモア
(社員・部下・出勤・投函・会議室、という擬人化語彙を UI 文言全体で貫く)。

- **グリッド**: `grid-template-columns: 280px 1fr 300px; gap: 16px;` / ページ余白 `padding: 12px 20px`。
  実測比率: 左 ≈17% / 中央 ≈62% / 右 ≈18%。
- **トップバー**: 高さ ≈90px。左に LIVE ピル (最上段) → タイトル → サブタイトルの縦積み。
  右端にスタッツ tile ×4 を横並び (上下センター)。
- **中央カラム**: ステージのベゼルが幅いっぱい。ベゼル比率はおよそ 3:2 (例: 930×630)。
  ベゼルの下、中央寄せでデータ源の注記 1 行 (小さいグレー文字 + code チップ)。
- **左右サイドバー**: 各カードは縦積み、`gap: 16px`。右の作業フィードは残り高さを使い内部スクロール。
- **レスポンシブ方針** (動画に根拠なし・提案):
  デスクトップファースト。`< 1280px` でサイドバーを 240px に縮小、`< 1024px` で
  「トップバー → ステージ → 指示センター → 社員詳細 → フィード」の 1 カラム縦積みに落とす。
  ステージは常に `max-width: 100%` で縮小し、内部は座標系ごとスケール (§6)。

## 4. タイポグラフィ

3 系統を使い分ける。すべて日本語グリフ必須。

1. **クロム層 (本文・カード)**: 端正で丸すぎない日本語サンセリフ。
   候補: **"Zen Kaku Gothic New"** (第一候補・穏やかで cozy に合う) / "M PLUS 1p" /
   fallback `"Hiragino Kaku Gothic ProN", "Noto Sans JP", sans-serif`。
   - 本文 13px / 行間 1.6。ヘルプ文・注記 11px。カード見出し 13px **bold** + 字間 +0.02em。
   - ページタイトル「🏢 せなおAIオフィス」: 22px / weight 800 / `--text-primary`。
   - スタッツ数値: 24px / weight 700 (等幅数字 `font-variant-numeric: tabular-nums` 推奨)。
2. **ステージ層 (ピクセル世界の中の文字)**: ビットマップ調。
   候補: **"DotGothic16"** (Google Fonts。日本語ドット文字の決定版) を吹き出し・名前チップ・
   看板系ラベルに。英字看板 "SENAO AI OFFICE" は **"Press Start 2P"** か画像に焼き込み。
   `font-smooth: never` 相当は不可能なので、小さめサイズ (10–12px) + `image-rendering: pixelated`
   の背景と合わせて馴染ませる。
3. **等幅 (パス・ブランチ・code チップ)**: `"IBM Plex Mono", "SF Mono", monospace`。
   社員詳細の作業場所パス、フッターの `~/.claude/projects/*.jsonl` チップ、
   吹き出し内の `<task-notification> <task-id>` などに 11px で使用。

禁止: Inter / Roboto / Arial をそのまま使わない (skill 原則)。ただし本 UI は日本語主体なので、
「日本語フォントの選定」が実質のタイポ判断。DotGothic16 の採用がステージらしさの 8 割を決める。

## 5. コンポーネント仕様

共通: カードは `background: var(--bg-card); border: 1px solid var(--border-card);
border-radius: 12px; box-shadow: var(--shadow-card); padding: 14–16px;`。
カード見出しは「絵文字 + 見出し語」+ 直下に `--border-strong` の 2px 罫線 (幅は文字分だけでなく全幅)。

### 5.1 トップバー

- **LIVEピル**: `border-radius: 999px; border: 1px solid var(--border-strong);
  background: #F7F1E3; padding: 4px 14px; font-size: 11px; letter-spacing: .08em;`
  先頭に緑ドット 8px (`--status-green`、`animation: pulse 2s infinite` 提案)。
  文言: `● LIVE — このMacの Claude Code`。
- **タイトル行**: 🏢 + 「せなおAIオフィス」。サブタイトルは 12px `--text-body`。
- **スタッツ tile ×4**: 各 `min-width: 64px; padding: 8px 12px;` の白カード (角丸 10px)。
  上段=数値 24px bold (カテゴリ色)、下段=`● ラベル` 10px (ドットはカテゴリ色、承認/質問のみ ❗)。
  順序: 作業中(緑) / 指示待ち(琥珀) / 承認・質問(赤) / 出勤(墨)。数値は 3 秒更新で変化する
  (観測: 4→2→1→4 など)。変化時に軽い scale ポップ (1→1.15→1, 200ms) を提案。

### 5.2 指示センター (左上カード)

- 見出し: `🎛 指示センター`。
- **宛先ドロップダウン**: 高さ 34px、角丸 8px、`border: 1px solid var(--border-strong)`。
  表示文言パターン: 未選択=`宛先を選択(キャラをクリック)` / 選択中=`動画編集ソフト(実行中)` /
  配達待ちあり=`動画編集ソフト(実行中・✉)` ← **キュー有無で ✉ が付く**。右端に ▼。
  ステージのキャラクリックと双方向連動 (クリックで宛先が入る)。
- **テキストエリア**: 高さ ≈72px、角丸 8px、placeholder =
  `例: キリのいいところで進捗をまとめてコミットして、残タスクを報告して` (muted色)。
- **投函ボタン**: 全幅 / 高さ 40px / 角丸 10px / 白文字 14px bold /
  `background: linear-gradient(180deg, var(--accent-brown), var(--accent-brown-deep))`。
  文言 `投函する (⌘+Enter)`。⌘+Enter で送信。hover で明度 +5%、active で 1px 沈む。
- **ヘルプ文** (11px, muted, 2行):
  `届くタイミング=待機中の社員なら数秒以内/作業中はターン終了直後。✉=配達待ち・❗=承認/質問待ち。`

### 5.3 送信履歴 (左下カード)

- 見出し: `📮 送信履歴`。新しい順の縦リスト。
- 各アイテム: 1 行目 = `00:51`(等幅 muted 11px) + **宛先名**(bold, `--accent-brown` 系の茶) +
  右端に状態チップ。2 行目以降 = 送った本文 12px `--text-body` (折返し)。
- 状態チップ: `✅ 配達済み` (ok色) / `✉ 配達待ち` (wait色)。`border-radius: 999px;
  padding: 2px 8px; font-size: 10px;`。配達確認で待ち→済みへ更新。
- 対話オーバーレイのボタン押下でも定型文がここへ積まれる (§5.7・§7)。

### 5.4 社員詳細 (右上カード)

- 見出し: `👤 社員詳細`。
- **未選択時**: カード中央に muted 12px で `オフィスのキャラをクリックするとここに詳細が出ます` の 2 行。
- **選択時**:
  - ヘッダ: アバター (キャラスプライトの顔まわり 40×40、`image-rendering: pixelated`) +
    名前 15px bold + 状態行 `● 作業中・2分前` (ドットは状態色。「・N分前」は状態継続時間)。
  - **キーバリュー表** (5行、各行間に hairline):
    `いま` = 現在タスク要約 (2行まで、折返し) /
    `作業場所` = cwd フルパス (等幅 11px、折返し) /
    `ブランチ` = `⎇ develop` (等幅) /
    `部下` = `なし` or `N名` (サブエージェント数) /
    `直近` = 直近アクティビティ 2〜3 件 (例: `編集中 package.json`)。
    ラベル列 ≈64px muted 11px、値列は残り幅 12px。
  - **ボタン**: 全幅アウトライン型 `✉ この社員に指示を出す`。
    `border: 1.5px solid var(--accent-gold); color: var(--accent-brown); background: #FBF6E9;
    border-radius: 10px; height: 36px;` 押下で指示センターの宛先に設定 + textarea へフォーカス。

### 5.5 作業フィード (右下カード)

- 見出し: `📋 作業フィード`。残り高さで `overflow-y: auto`。
- 各アイテム: 左に相対時刻 (`15秒前` / `3分前`、11px muted、幅 ≈48px 固定) +
  **セッション名** (12px bold 茶、長い場合 `20260622 Che…` と省略) + 本文 12px。
  本文には 💬 (発話要約) / ✅ (完了) 等の絵文字が接頭に付くことがある。3 行程度で `…` 省略。
- 新着は**先頭に挿入**され、既存は下へ押し出される。相対時刻は 3 秒ごとに再計算
  (観測: 15秒前→18→22→26秒前と同一アイテムが加齢していく)。
- 新着挿入時のモーション提案: `opacity 0→1` + `translateY(-4px)→0` 200ms。

### 5.6 ステージ額装 (ベゼル) とフッター注記

- ベゼル: `background: var(--stage-bezel); border-radius: 18px; padding: 14px;` +
  大きめの接地影 `0 18px 40px rgba(61,46,30,.18)`。内側にピクセルアートを敷く。
- ベゼル下の注記 (中央寄せ 11px muted):
  `データ源: ~/.claude/projects/*.jsonl (読み取りのみ・完全ローカル)・背景/キャラ=GPT Image 2.0製 (再生成 assets_gen.py)・表示名: office_config.json`
  — パス・ファイル名部分は code チップ (`background: #EFE7D2; border-radius: 4px;
  padding: 1px 6px; font-family: mono`)。

### 5.7 対話オーバーレイ (ステージ下部・最重要インタラクション)

- 位置: ステージ内側の下端に absolute 配置。幅 ≈ ステージ内幅 − 24px、高さ ≈140–170px (可変)。
- 見た目: `background: var(--overlay-bg); border: 2px solid var(--overlay-border);
  border-radius: 10px; padding: 12–16px;` RPG の会話ウィンドウ様式。
- **左**: 対象キャラの立ち絵スプライト (≈64×88px, pixelated) + 直下に名前 (DotGothic16 11px 白)。
- **本文** (クリーム白 13px、DotGothic16 か通常サンセリフ+ステージ調どちらでも可・原画は角ゴシック):
  - 1 行目: `動画編集ソフトです!いまは「実行中 Verify MCP port closed on ai startup and …」です。`
    — タスク名の鉤括弧内は **金色 bold**。
  - 2 行目 (承認待ち時): `⚠ 承認待ちかも: 「…」の実行が約2分止まっています。ターミナルの許可ダイアログを確認してください。▼`
    — ⚠ と本文を琥珀色で。
  - 追記行: ボタン押下ごとに `📮 投函しました!手が空き次第すぐ伝えます。` が**下に積まれる**。
  - **逐字表示 (タイプライター) 効果あり** (frame_00 で文中まで、frame_01 で全文表示を確認)。
    1 文字 15–25ms 程度。
- **アクションボタン列** (本文の下、左寄せ横並び、gap 8px):
  `✅ 承認・進めて` / `🔴 一旦ストップ` / `📋 報告して` / `👋(☁) 自由に指示` —
  ダーク地 (`#2A1D10`)、細枠 (`#5A4425`)、角丸 8px、白文字 12px、絵文字だけカテゴリ色
  (緑/赤/白/金)。hover で枠が `--overlay-border` に。
  右端に独立して `✕ 閉じる` (同スタイル)。
- ボタンの意味: 承認・進めて / 一旦ストップ / 報告して は**定型指示文を投函** (送信履歴に載る。
  例: 承認→`直前の確認・質問について承認します。そのまま進めてください。`、
  報告→`今の進捗と残タスクを3行で報告してください。`)。自由に指示 = 指示センターへフォーカス移動。
- **注記行** (最下部 10px グレー):
  `※ターミナルの許可ダイアログ自体はここからは押せません。ボタンの指示は相手が手を止めた瞬間(またはターン終了時)に届きます。`
- 開閉: キャラクリックで開く (対象切替も可) / ✕ で閉じる。閉時はステージ全面が見える。

### 5.8 ステージ内オブジェクト (吹き出し・チップ類)

- **名前チップ** (キャラ足元/下): `--bubble-bg` 地、白文字 10px (DotGothic16)、角丸 4px、
  padding 2px 6px。例 `動画編集ソフト` `ブログ編集部` `制作本部(works) 9号`。
- **状態吹き出し** (キャラ頭上): ダークツールチップ。1 行目 = 名前 (金 `--bubble-name`)、
  2 行目以降 = `💭 考え中…` `📥 指示を受領` `結果を確認して次の一手` 等 10–11px。
  等幅の生ログ断片 (`<task-notification> <task-id> biubqrh8i</t…`) がグレーで混ざることもある。
  下向きの小さな三角しっぽ付き。同時に複数キャラ分表示される。
- **承認待ちアラート吹き出し**: `--alert-bg` 地 + `--alert-border` 枠。
  1 行目 = `動画編集ソフト — 確認まち!` (金)。
  2 行目 = `⚠ 「実行中 Verify MCP port closed on ai startup and …」が約2分停止中`
  — **停止時間は経過でカウントアップ** (観測: 約2分→約4分→約5分)。
  3 行目 = `▶ クリックして対応` (赤みの明るい文字)。クリックで対話オーバーレイが開く。
  通常吹き出しより優先度高 (z 上位・サイズ大)。ゆっくり明滅 (opacity 0.9⇄1.0, 1.5s) を提案。
- **部屋ラベルチップ**: `🏷 会議室 — Workflow部隊` / `🏷 休憩ラウンジ` — ダークチップ、
  部屋の上辺に固定配置。
- **ホワイトボード**: 会議室内。`本日: 作業4・待機10` — **トップバーのスタッツと連動して
  数値が書き換わる** (観測: 作業2・待機12 / 作業1・待機13)。
- **ミニオン (部下) スプライト**: 会議室の床に小さなピンクの生き物 (子豚風) が複数出現。
  リーダー格キャラの吹き出しに `👥 部下95名` などの数が出る (観測: 95→93→91名と減少 =
  サブエージェント数のライブ反映)。数が多くても描画は数匹に丸める。

## 6. 中央ピクセルアートステージの実装方針 (最重要・最難関)

### 6.1 全体構成 = 「1枚絵の背景 + DOMオーバーレイ」方式 (推奨)

原画の作りもこの方式 (フッター注記より: 背景/キャラ = GPT Image 2.0 製の生成画像、
表示名は office_config.json、フィードに「会話ウィンドウの z-index」「walk frame 生成」への言及あり)。

- **レイヤ0 — 背景 1 枚絵** (`office_bg.png`, 内部解像度例 1860×1260):
  部屋・壁・床・家具 (デスク・モニター・本棚・楕円テーブル・椅子・ソファ・観葉植物・時計・
  看板・窓・ドア・ウォーターサーバー・絵画) をすべて焼き込んだ静止画。
  `image-rendering: pixelated; width: 100%;` で表示。
- **レイヤ1 — キャラスプライト**: 透過 PNG をキャラごとに absolute 配置。
  最低 2 ポーズ (着席/立ち)。余力があれば歩行 2–4 フレーム (原画は walk frame を持つ)。
  chibi 頭身 (約 2.5 頭身)、サイズ ≈48×64px (内部解像度基準)。
- **レイヤ2 — 名前チップ** (キャラ下端に追従)。
- **レイヤ3 — 状態吹き出し** (キャラ上端に追従。画面端でクランプ)。
- **レイヤ4 — 承認待ちアラート吹き出し** (通常吹き出しより上)。
- **レイヤ5 — 対話オーバーレイ** (ステージ下部固定)。
- キャラ同士の前後関係: **y 座標昇順で z-index を振る** (下にいるキャラほど手前)。
  チップ・吹き出しは親キャラの z を継承 +1。

### 6.2 座標系とスケーリング

- ステージ内部を**固定の論理座標系** (例 1860×1260) で扱い、外側コンテナに合わせて
  `transform: scale()` するか、全配置を % で持つ。**席・部屋は設定ファイルで宣言**する:

```json
// office_config.json (例)
{
  "rooms": {
    "works":   { "label": "制作本部", "seats": [ {"x": 410, "y": 300}, {"x": 545, "y": 300}, … ] },
    "meeting": { "label": "会議室 — Workflow部隊", "spots": [ {"x": 940, "y": 290} ] },
    "lounge":  { "label": "休憩ラウンジ", "spots": [ {"x": 1010, "y": 560}, {"x": 1160, "y": 560} ] }
  },
  "whiteboard": { "x": 1030, "y": 175 },
  "displayNames": { "session-uuid": "動画編集ソフト" }
}
```

- 部屋レイアウト (原画準拠):
  - **左〜中央 2/3 = 制作本部 (works)**: デスク 4 列 × 4 行前後のグリッド。上の壁に
    本棚×2・窓×2 (外に緑) ・「SENAO AI OFFICE」看板・掛け時計。
  - **右上 = 会議室**: ティール床 + ガラスパーティション、楕円テーブル + 椅子 8 脚、
    奥にホワイトボード、リーダーが立つ。Workflow 系 (サブエージェント持ち) セッションを配置。
  - **右下 = 休憩ラウンジ**: 市松床、赤ソファ 2 台、ローテーブル、ウォーターサーバー、絵画。
    **待機の長い社員がここへ移動して座る** (観測: 待機化した制作本部キャラがラウンジに出現)。
  - 境界に木の壁とドア (制作本部→右エリアの通用口)。

### 6.3 素材の入手戦略 (現実的な指針)

1. **本命 = 画像生成**: 背景 1 枚 + キャラ数種 (色違い/服違いで 8–12 バリエーション) を
   画像生成モデルで作る。プロンプト要点: "top-down 2D pixel art office, warm brown brick walls,
   wooden floor, 16-bit RPG interior, cozy amber lighting"。キャラは
   "chibi pixel art office worker sprite, transparent background, seated pose / standing pose"。
   生成物は必ず幅を 2 の倍数に整え、`image-rendering: pixelated` で拡大。
   再生成スクリプト (`assets_gen.py` 相当) を用意し、素材を差し替え可能にする。
2. **フォールバック = CSS/絵文字プレースホルダー**: 素材が揃うまでの間、
   背景は `--px-*` 変数の矩形 + `repeating-linear-gradient` (床板・レンガ・市松) で部屋を描き、
   キャラは絵文字 (🧑‍💻) + 色付き座布団 div で代替可能。**レイアウト・座標系・吹き出し・
   オーバーレイのロジックは素材と独立に完成させられる**ので、先にこちらで組む。
3. タイルマップ + スプライトシート (Phaser / PixiJS / canvas 自前) は自由度最大だが、
   このUIは「マップは静的・動くのはキャラと吹き出しだけ」なので**過剰**。DOM で足りる。

### 6.4 描画の質感

- 拡大は整数倍を優先 (1.5 倍などは にじみの元。scale は 2 倍素材を用意して吸収)。
- ステージ内のあらゆる画像に `image-rendering: pixelated`。
- 室内照明の暖かさは背景画像に焼き込み。CSS 側では `filter` を使わない (色ズレ防止)。
- 夜/昼の照明変化は動画からは**確認できず** — 実装しない (YAGNI)。窓明かりは常時暖色。

## 7. 動的挙動 / モーション

データは **3 秒ごとのポーリング**で全面更新。フレーム差分から観測した挙動:

| 挙動 | 観測根拠 (フレーム) | 実装仕様 |
|---|---|---|
| スタッツ数値の変化 | 作業中 4→2→1→4 / 指示待ち 10→12→13→10 (f00→f05→f07→f09) | 3秒ポーリングで再計算。変化時に数字を scale ポップ |
| ホワイトボード連動 | `本日: 作業4・待機10`→`作業2・待機12`→`作業1・待機13` | スタッツと同一ソース。テキストのみ書換え |
| 社員状態の遷移 | 詳細カード `作業中・2分前`→`指示待ち・4分前` (f06) | state + 経過分の再計算 |
| 承認待ちアラートの加齢 | `約2分停止中`→`約4分`→`約5分` (f00→f06→f07) | 停止開始時刻から毎更新で分数再計算 |
| キャラの移動 | 動画編集ソフトが前列席→左の立ち位置へ (f05); 待機社員がラウンジ出現 (f05,f09) | 状態変化で目標スポットへ移動。歩行フレームがあれば 2–4 コマで歩かせ、なければ 300ms フェード転移 |
| ラウンジ占有 | f00 は無人 → f05/f09 に 3 名 (制作本部(works), ブログ編集部, 9号) | 待機 N 分超の社員をラウンジ spot へ割当 |
| 会議室ミニオン | 小ピンク生物が床に出現、`部下95名`→`93名`→`91名` (f05–f08) | subordinates>0 のとき数匹描画 + 吹き出しに実数 |
| 対話の逐字表示 | f00 で文中断片 → f01 で全文 | typewriter 15–25ms/字。更新で本文が変わったら再タイプ |
| ボタン→投函→追記 | f03: `投函しました!` ×2 行追記 + 送信履歴に 00:51 ×2 (配達待ち) | ボタン押下で (1) outbox に定型文 push (2) ダイアログへ確認行 append |
| 配達状態チップ | `✉配達待ち` と `✅配達済み` が混在 (f03–f08) | セッションへの実配達確認で待ち→済み |
| ドロップダウンのキュー表示 | `(実行中)`→`(実行中・✉)` (f03) | 宛先に未配達があれば ✉ を付す |
| フィードのストリーム | 新着が先頭挿入、相対時刻が加齢 (全フレーム) | 先頭 insert + fade/slide 200ms。時刻は毎更新再計算 |
| 吹き出しの出現/消滅 | 各フレームで吹き出しの組合せが変化 | 状態変化イベントで表示、一定時間 or 状態解消で消す。opacity 150ms |

モーション原則: **芝居はステージ内に集中させ、クロム層は静かに保つ**。
クロム側のアニメは「数値ポップ」「フィード挿入」「ボタン hover」の 3 つだけ。
ステージ側は吹き出し・タイプライター・キャラ移動・アラート明滅で「生きている感」を出す。
`prefers-reduced-motion` ではタイプライターと明滅を無効化して即時表示。

## 8. 実装ガイダンス

### 8.1 推奨スタック

- **フロント**: Vite + **React** (または Vue。どちらでも成立)。状態は 3 秒ポーリング前提なので
  Redux 等は不要 — `useState` + `setInterval(fetchState, 3000)` で足りる。
  ステージは通常の DOM (絶対配置 div + img)。canvas 不要 (§6.3)。
- **バックエンド**: ローカル専用の小さな HTTP サーバー (Python: FastAPI or 標準ライブラリ。
  原画の実装も `_office_server.py` という Python サーバー)。責務:
  1. `~/.claude/projects/*/*.jsonl` を**読み取り専用**で tail し、セッション状態・フィードを構築
  2. `GET /api/state` → 下記 JSON を返す
  3. `POST /api/instruct` → 指示をキューに積み、対象セッションへ配達 (配達成否を状態に反映)
- 完全ローカル動作・外部送信なし、を UI 注記どおり守る。

### 8.2 データモデル (API レスポンス例)

```jsonc
{
  "stats": { "working": 4, "waiting": 10, "approval": 1, "present": 17 },
  "sessions": [
    {
      "id": "6f2c…",
      "displayName": "動画編集ソフト",          // office_config.json で上書き可
      "department": "works",                    // works | workflow | lounge 配置根拠
      "state": "approval",                      // working | waiting | approval
      "stateSince": "2026-07-13T00:49:12+09:00",
      "currentTask": "実行中 Verify MCP port closed on ai startup and …",
      "cwd": "/Users/shotaotobe/project/develop/動画編集ソフト",
      "branch": "develop",
      "subordinates": 0,                        // サブエージェント数 (会議室ミニオンの元数)
      "recent": ["実行中 Verify MCP port closed…", "編集中 package.json"],
      "stall":  { "since": "…", "reason": "permission-dialog" },  // approval 時のみ
      "seat": { "room": "works", "index": 3 },  // サーバー or クライアントで割当
      "say": "動画編集ソフトです!いまは「…」です。"   // 対話オーバーレイ用の台詞
    }
  ],
  "feed": [
    { "ts": "2026-07-13T00:54:41+09:00", "session": "20260622 CheckPointer",
      "kind": "note", "text": "この予約ウェイクアップの用件は**処理済み**です — Workflo…" }
  ],
  "outbox": [
    { "ts": "00:51", "to": "動画編集ソフト", "status": "pending",   // pending | delivered
      "text": "直前の確認・質問について承認します。そのまま進めてください。" }
  ]
}
```

- 状態判定の目安: jsonl の最終イベントが実行系 → `working` / ターン終了後入力待ち → `waiting` /
  許可ダイアログ等で N 秒停止 → `approval` (+ stall.since)。
- 定型指示文 (対話ボタン): 承認=`直前の確認・質問について承認します。そのまま進めてください。` /
  ストップ=`一旦手を止めてください。` (推定・要調整) / 報告=`今の進捗と残タスクを3行で報告してください。`

### 8.3 実装順序の提案

1. クロム層 (トップバー + 3 カラム + 全カード) をモックデータで完成 → 確認: 静的画面がフレームと並べて概ね一致
2. ステージの座標系 + CSS プレースホルダー背景 + キャラ配置 + 吹き出し → 確認: office_config の席にキャラが立つ
3. 対話オーバーレイ + タイプライター + ボタン→送信履歴の流れ → 確認: クリック往復が動く
4. 3 秒ポーリング + 実 jsonl 読取り → 確認: 実セッションでスタッツ・フィードが動く
5. 生成素材 (背景 1 枚 + キャラ数種) に差替え → 確認: 原画フレームと見比べて質感一致
